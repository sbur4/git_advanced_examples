1. Переключение на существующую ветку:
git switch branch_name
Здесь branch_name - это имя существующей ветки, на которую вы хотите переключиться. Например, git switch main переключит вас на ветку "main".

2. Создание и переключение на новую ветку:
git switch -c new_branch_name
Эта команда создает новую ветку с именем new_branch_name и автоматически переключает вас на нее. Например, git switch -c feature-branch создаст новую ветку "feature-branch" и переключит вас на нее.

3. Переключение на предыдущую ветку:
git switch -
Эта команда переключит вас обратно на ветку, с которой вы только что переключились. Это полезно, если вы хотите быстро вернуться на предыдущую ветку после выполнения каких-то действий на другой ветке.

4. Интерактивное переключение с автодополнением имени ветки:
git switch -c
Эта команда предоставит вам интерактивный интерфейс для выбора существующей ветки или создания новой с автодополнением имени.

5. Создание новой ветки на основе удаленной ветки и переключение на нее:
git switch -c new_branch_name origin/remote_branch_name
Эта команда создаст новую ветку new_branch_name на основе удаленной ветки remote_branch_name и автоматически переключит вас на новую ветку.

Это основные варианты использования команды git switch. Команда git switch предоставляет удобный и безопасный способ управления ветками в Git и уменьшает риск ошибок при переключении между ветками.
