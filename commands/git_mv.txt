1. Перемещение файла или директории в другое место и задание нового имени:
git mv oldfile.txt newfile.txt
git mv old_directory new_directory
Этот вариант команды git mv перемещает oldfile.txt в newfile.txt и обновляет индекс Git для отслеживания этого перемещения.

2. Переименование файла или директории без изменения местоположения:
git mv oldfile.txt newfile.txt
Этот вариант команды git mv позволяет переименовать файл oldfile.txt в newfile.txt, но оставить его в том же каталоге.

3. Перемещение и переименование файлов с заданием пути к новому местоположению:
git mv old_directory/file.txt new_directory/newfile.txt
Этот вариант позволяет переместить и переименовать файл из одного каталога в другой, указав полные пути к новому местоположению и новому имени файла.

4. Перемещение и переименование с использованием относительных путей:
git mv ../old_directory/file.txt new_directory/newfile.txt
В этом случае вы можете использовать относительные пути для перемещения и переименования файлов.

5. Перемещение и переименование с форсированием (force):
git mv -f oldfile.txt newfile.txt
Флаг -f или --force используется для принудительного перемещения или переименования, даже если Git может считать, что файлы по-прежнему существуют в старых путях.

После использования git mv, не забудьте сделать коммит, чтобы зафиксировать изменения в репозитории.
